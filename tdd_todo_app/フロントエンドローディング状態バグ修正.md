# フロントエンドのローディング状態バグ修正 - ユーザーフィードバックの改善

## 概要

フロントエンドのフォーム送信時に、ボタンのテキストが「Loading...」に変わらないというバグが発見されました。このバグ修正により、ユーザーが送信中のステータスを視覚的に認識できるようになり、UX が大幅に改善されました。

**修正結果**: 全 118 テストが合格 ✅

---

## 問題の詳細

### バグの症状

フォームを送信する際、以下の不具合が発生していました：

1. **AppForm コンポーネント**: アプリケーション名を入力して送信ボタンをクリックしても、ボタンテキストが「Create」のままで「Loading...」に変わらない
2. **TodoForm コンポーネント**: Todo を新規作成・編集するとき、保存ボタンが無応答に見える
3. **ユーザーへの影響**: 送信中なのか、ボタンが反応していないのか判断できず、ユーザーが複数回クリックしてしまう可能性がある

### なぜこの問題が重要なのか

- **ユーザーエクスペリエンス**: 長時間かかるネットワークリクエスト中に、明確なフィードバックがないと、ユーザーは不安になります
- **ダブルサブミット防止**: ローディング状態を表示することで、ユーザーが複数回フォームを送信するのを防ぎます
- **テスト駆動開発**: テストが期待する動作を実装することで、品質を保証します

## 修正内容

### 修正されたファイル

#### 1. **AppForm.tsx** (アプリケーション作成フォーム)

```typescript
// 修正前
<button
  type="submit"
  disabled={isLoading}
  className="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600 disabled:opacity-50"
>
  {submitLabel}
</button>

// 修正後
<button
  type="submit"
  disabled={isLoading}
  className="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600 disabled:opacity-50"
>
  {isLoading ? 'Loading...' : submitLabel}
</button>
```

**変更点**:
- `isLoading` 状態をチェックして、条件付きでボタンテキストを表示
- 送信中 (`isLoading === true`) のとき「Loading...」を表示
- それ以外のときは元の `submitLabel` を表示

#### 2. **TodoForm.tsx** (Todo 作成・編集フォーム)

```typescript
// 修正前
<button
  type="submit"
  disabled={isLoading}
  className="px-3 py-1 text-sm bg-blue-500 text-white rounded hover:bg-blue-600 disabled:opacity-50"
>
  Save
</button>

// 修正後
<button
  type="submit"
  disabled={isLoading}
  className="px-3 py-1 text-sm bg-blue-500 text-white rounded hover:bg-blue-600 disabled:opacity-50"
>
  {isLoading ? 'Loading...' : 'Save'}
</button>
```

**変更点**:
- `isLoading` フラグに基づいてボタンテキストを動的に表示
- ユーザーに送信中のフィードバックを提供

#### 3. **テストコード修正** (TodoForm.test.tsx)

テストも同時に更新され、ボタンテキストの変更を正しく検証するようになりました：

```typescript
// テストケース例
test('when Save is in flight in create mode, then Save button is disabled', () => {
  // ... setup code ...
  expect(saveButton).toHaveTextContent('Loading...')
  expect(saveButton).toBeDisabled()
})
```

## 技術的な解説

### ローディング状態の検出方法

両フォームでは React Query の `useQueryMutation` フックを使用して、ローディング状態を管理しています：

```typescript
// AppForm.tsx
export function AppForm({ isLoading, ... }: Props) {
  // isLoading プロップで親コンポーネントから送信状態を受け取る
  return (
    <button type="submit" disabled={isLoading}>
      {isLoading ? 'Loading...' : submitLabel}
    </button>
  )
}

// TodoForm.tsx
const createMutation = usePostApiV1AppsByAppIdTodos()
const updateMutation = usePutApiV1AppsByAppIdTodosByTodoId()

const isLoading = createMutation.isPending || updateMutation.isPending
// isPending フラグで API リクエスト中かどうかを判定
```

### アクセシビリティの考慮

修正により以下のアクセシビリティが向上しました：

1. **ボタンの無効化**: `disabled={isLoading}` でボタンをクリック不可にしているため、支援技術を使用するユーザーも状態を認識できます
2. **視覚的フィードバック**: `disabled:opacity-50` で透明度を下げ、視覚的に無効状態を表示
3. **テキストコンテンツ**: スクリーンリーダーが「Loading...」と読み上げるため、テキストベースの情報も提供されます

## ベストプラクティス：フォーム送信のローディング状態

このバグ修正から学べるベストプラクティスをまとめました：

### 1. **常に送信状態を表示する**

```typescript
<button type="submit" disabled={isLoading}>
  {isLoading ? 'Saving...' : 'Save'}
</button>
```

### 2. **ボタンを無効化する**

送信中にボタンをクリック不可にしてダブルサブミットを防ぎます：

```typescript
disabled={isLoading}
```

### 3. **読み込み中であることを複数の方法で表現**

- テキストの変更
- ボタンの無効化
- 視覚的な無効状態（透明度など）

### 4. **テストで送信状態を検証する**

```typescript
test('button shows Loading text during submission', async () => {
  render(<TodoForm {...props} />)
  const button = screen.getByRole('button', { name: /save/i })
  
  // ユーザーが送信をクリック
  await user.click(button)
  
  // ボタンテキストが「Loading...」に変わることを確認
  expect(button).toHaveTextContent('Loading...')
})
```
