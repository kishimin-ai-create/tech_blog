## バグ修正

### 修正対象（3件のローディング状態バグ）

フロントエンド開発初期段階で、ローディング状態の表示に関する3つのバグが発見されました。いずれも「ローディング中は正しいボタンテキストを表示する」という仕様実装の不備でした。

#### 1. AppDetailPage.tsx - ReferenceError（変数宣言順序）

**問題：** `app` 変数が使用される前に宣言されていなかったため、`ReferenceError` が発生していました。

**根本原因：** 変数 `app` が参照される箇所より後ろで宣言されていた。

**修正内容：**
```typescript
// 修正前（エラー）
const appId = app?.id;  // ← 使用
const app = response.data; // ← 宣言が後

// 修正後（正常）
const app = response.data; // ← 先に宣言
const appId = app?.id;     // ← 使用
```

**影響：** AppDetailPage でアプリの詳細表示時にクラッシュが発生していました。

---

#### 2. AppForm.tsx - ローディング状態ボタンテキスト表示

**問題：** `isLoading` が `true` の際、ボタンテキストが「Loading...」と表示されていませんでした。

**根本原因：** ローディング状態時の条件分岐でテキストを更新する処理が実装されていなかった。

**修正内容：**
```typescript
// 修正前
<button disabled={isLoading}>
  {mode === 'create' ? 'Create' : 'Update'}
</button>

// 修正後
<button disabled={isLoading}>
  {isLoading ? 'Loading...' : (mode === 'create' ? 'Create' : 'Update')}
</button>
```

**影響：** ユーザーがフォーム送信中かどうかを視覚的に判断できず、UX が低下していました。

---

#### 3. TodoForm.tsx - ローディング状態ボタンテキスト表示

**問題：** 同じく `isLoading` 時にボタンテキストが更新されていませんでした。

**根本原因：** AppForm.tsx と同じ実装漏れ。

**修正内容：**
```typescript
// 修正前
<button disabled={isLoading}>
  {todo ? 'Save' : 'Create'}
</button>

// 修正後
<button disabled={isLoading}>
  {isLoading ? 'Loading...' : (todo ? 'Save' : 'Create')}
</button>
```

**テスト検証：** 修正後、TodoForm.test.tsx のテスト内容も更新され、ローディング状態時の表示を正確に検証するようになりました。

```typescript
// テスト例
expect(button).toHaveTextContent('Loading...');
```

**影響：** Todo フォーム（作成・編集）で送信状態が不明瞭でした。

## 検出された問題

### TypeScript コンパイルエラー 2件

コードレビュー工程で、以下の2件のTypeScriptコンパイルエラーが検出されました。いずれも **未使用のインポート** に関する問題です。

#### Issue 1: AppList.test.tsx - 未使用の `vi` インポート

**ファイル：** `frontend/src/features/app-list/components/AppList.test.tsx`

**エラーメッセージ：**
```
'vi' is declared but its value is never read.
```

**原因：** Vitest の `vi` 関数がインポートされているが、テストコード内で実際には使用されていなかった。

**コード例（修正前）：**
```typescript
import { describe, it, expect, vi } from 'vitest';  // ← vi はここにのみ

describe('AppList', () => {
  it('renders apps', () => {
    // ... テストコード
    // vi は使用されていない
  });
});
```

---

#### Issue 2: AppListPage.test.tsx - 未使用の `beforeEach` インポート

**ファイル：** `frontend/src/features/app-list/pages/AppListPage.test.tsx`

**エラーメッセージ：**
```
'beforeEach' is declared but its value is never read.
```

**原因：** Vitest の `beforeEach` フックがインポートされているが、テストコード内で使用されていなかった。

**コード例（修正前）：**
```typescript
import { describe, it, expect, beforeEach } from 'vitest';  // ← beforeEach はここにのみ

describe('AppListPage', () => {
  // beforeEach は定義されていない
  
  it('shows app list', () => {
    // ... テストコード
  });
});
```

---

### エラーの影響

これらの未使用インポートは：
- **TypeScript strict mode では許容されない** 違反
- ビルドを失敗させる可能性がある
- 本番コードのコンパイルをブロック
- 将来のメンテナンス時に混乱を招く
